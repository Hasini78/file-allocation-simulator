Contiguous Allocation

Sequential Blocks
Files stored in consecutive blocks. Fast, but vulnerable to fragmentation.
Disk Storage (32 Blocks)
Directory Table
FileStart BlockLengthBlocks Used
How it works
1.OS searches for N contiguous free blocks
2.Marks them as used in the bitmap
3.Writes file data sequentially
4.Directory stores only: start + length
🔗

Linked Allocation

Chain of Blocks
Each block points to the next. No fragmentation, but slow random access.
Disk Storage (32 Blocks)
File Chains
Key Insight
📌 Directory stores only the FIRST block
🔗 Each block: data + pointer to next
🏁 Last block pointer = NULL
⚠️ Random access = slow (must traverse)
📋

Indexed Allocation

Index Block
An index block maps all data blocks. Best of both worlds — flexible and fast.

1. Directory → index block #
2. Index block → data block #
3. Read data block directly
